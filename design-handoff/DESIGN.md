# スイムランク デザインブリーフ

マスターズ水泳の「年齢別資格表」（20級〜1級）に対して、自分のタイムが何級かを判定し、級を軸に種目・距離を横断比較できるモバイルファーストPWA。

- **実物**: https://kanekop.github.io/swimrank/ （スマホで開くのが最良。ホーム画面に追加可）
- **リポジトリ**: https://github.com/kanekop/swimrank （React 18 + TypeScript + CSS Modules + Vite）
- **開発**: Yoshimune Kaneko

## このパッケージの中身

| ファイル | 内容 |
|---|---|
| `screens/*.html` | 実アプリのDOMとコンパイル済みCSSをそのままキャプチャした忠実なモックアップ。ブラウザで開くだけで見える（375px幅想定） |
| `tokens.html` | デザイントークン一覧（色・文字・余白・角丸） |
| `screens/_app.css` | 全画面共通のコンパイル済みCSS（クラス名はCSS Modulesのハッシュ付き） |

**注意**: モックアップのクラス名（`_badge_jnk5y_1` など）はビルド生成物です。デザイン改善の成果物はクラス名に依存しない形（改善したHTML/CSS、またはトークンと画面ごとの変更指示）で返してもらえれば、開発側でReactに反映します。

## ユーザーと利用場面

- 日本のマスターズスイマー（18歳〜90歳超、中心は40〜70代）
- プールサイドや練習後にスマホで片手操作。老眼でも読めるサイズ感が重要
- 核となる欲求: 「自分のタイムは何級か」「隣の泳法・距離では同じ級が何秒か」「スタイル1〜4（得意順）が一目でわかるか」

## 画面フロー

```
初回: onboarding → home
通常: home ⇄ kentei（タイム入力）→ result（級バッジ+ラダー）→ cross（横断比較）
タブ: [ホーム] [ランク判定] [ランク内で比較] [設定]
```

- `onboarding.html` — 初回セットアップ。名前・生年月日・性別。水泳年齢のライブプレビュー
- `home.html` — 挨拶 + マイ記録（級の高い順 = スタイル1〜4が可視化される）
- `kentei.html` — 種目チップ → 距離チップ → 分/秒/百分の一の3分割タイム入力
- `result.html` — 級バッジ（hero）+「あと1.36秒で9級」+ 20級→級外の全ラダー（自分の行ハイライト★）
- `cross.html` — 級ステッパー（−8級+）+ 同じ距離での4泳法比較 + 同じ泳法での全距離比較。✓は保存記録
- `settings.html` — プロフィール編集・データ管理・アプリ情報

## 表示状態のバリエーション（モックにない状態）

- 級外（1級より遅い）: バッジがグレー（--color-disabled）で「級外」、「あと X.XX秒 で 1級」
- 20級（最高位）: 「20級基準より X.XX秒 速い！」（アクセント色）
- データなし: 90歳以上の区分など「この年齢区分の基準データがありません」
- 推定値: 元データ補修セルはタイムに `*` + 脚注
- 記録ゼロのホーム: 空状態メッセージ + 判定へのCTA
- 18歳未満: 注記バナー
- ダークモード: prefers-color-scheme で全トークン差し替え（tokens.html参照）

## デザイン上の制約（維持してほしいもの）

1. タップ目標 48px 以上、下部タブバー + 親指リーチ優先のアクション配置
2. タイムは等幅フォント（桁ズレ防止）
3. 級バッジ・ラダーの自分の行ハイライトは機能の核。視認性を下げない
4. 依存追加なしで実装可能な範囲（CSSのみ、または軽量なReact変更）が反映しやすい
5. 日本語UI。ターゲット年齢層を考慮したコントラストと文字サイズ

## ブラッシュアップしてほしい観点（自由に提案歓迎）

- 全体のビジュアルアイデンティティ（現状は無難な青系。「水泳らしさ」「級が上がる楽しさ」の演出余地）
- 結果画面の感情設計（級が出た瞬間の喜び、次の級への動機づけ）
- 横断比較テーブルの情報密度と読みやすさ
- タブバーのアイコン（現状は絵文字。統一感のあるアイコンセットへ）
- ダークモードの調整
